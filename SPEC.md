# AI 客服助手 - 项目规格说明书 v2

## 项目状态：规划完成，待启动 Phase 1

---

## 1. 产品定义

**产品名：** Crypto Support MVP
**形态：** OpenClaw 内部 Demo Agent（Phase 1）；面向客户的 AI 客服产品（Phase 2+）
**一句话定位：** 一个会"理解问题、查状态、找原因、告诉你下一步怎么做"的加密平台支持型 Agent。

**不是：** FAQ 机器人 / 交易机器人 / 投资建议机器人 / 替代人工裁决的系统

---

## 2. 支持范围

- **平台：** OKX / Binance
- **用户类型：** 新注册用户 / 资产操作用户 / 规则查询用户
- **问题类型：** 解释型 / 查询型 / 诊断型 / 引导型

---

## 3. MVP 四大场景（Phase 1 必须闭环）

| # | 场景 | 核心能力 |
|---|------|---------|
| 1 | 知识问答 | 币链参数/费率/规则/公告 |
| 2 | KYB/KYC 资料完整性检查 | 缺项识别/退回原因/补交流程 |
| 3 | 提币未到账排查 | 定位订单→查平台→查链上→归因→建议 |
| 4 | 钱包/网络状态查询 | 某币某链充提开关/维护公告/替代网络 |

---

## 4. Agent 核心能力闭环

1. 意图识别 → 2. 上下文记忆 → 3. 槽位补全 → 4. 工具调用 → 5. 结果解释 → 6. 边界控制

---

## 5. 回答格式（固定）

```
1. 结论
2. 依据（附来源）
3. 下一步
4. 工单摘要（如有）
```

---

## 6. 严格不做

- 不执行交易 / 不读取账户私有资产 / 不发起高风险操作
- 不承诺审核结果 / 到账时间 / 恢复时长
- 不编造规则，必须基于工具结果或知识库

---

## 7. Phase 1 验收标准

- 用户提问 → AI 3 轮内给出有来源的诊断或指引
- 预填工单 JSON 结构完整率 ≥ 90%
- 框架可复用到其他场景（换知识库 + 诊断逻辑，不改架构）
