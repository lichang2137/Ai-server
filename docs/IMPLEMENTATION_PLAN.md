# AI 客服助手 - Phase 1 实施计划

## 目标
搭建 AI 客服助手 MVP，优先落地 P0 六大场景的知识问答 + 诊断排障核心链路。

---

## Phase 1 工作包

### WP1：基础设施
- [ ] 项目代码骨架（FastAPI / Next.js 或其他技术栈待确认）
- [ ] RAG 知识库框架选型（Chroma / Elasticsearch / 其他）
- [ ] 文档解析管线（帮助中心文章 → 向量化）
- [ ] 飞书知识库接入（Bitable 作为知识存储层）

### WP2：知识库搭建（L1-L3）
- [ ] L1 通用规则 FAQ 录入（登录/安全/KYC/地区限制/风控）
- [ ] L2 产品规则录入（现货/合约/杠杆/理财）
- [ ] L3 参数库录入（币链参数/最小量/memo/确认数/手续费）
- [ ] L4 运营动态接入（公告/钱包维护）
- [ ] L5 故障排障库录入（P0 场景对应的诊断逻辑）
- [ ] L6 工单知识录入（字段/团队边界/SLA）

### WP3：核心对话引擎
- [ ] 意图识别模块（P0 6个场景 + 通用兜底）
- [ ] 多轮对话管理（状态机 / 会话记忆）
- [ ] RAG 检索 + 生成管线
- [ ] 来源溯源（返回引用文档片段 + 链接）

### WP4：P0 场景诊断流
- [ ] 充值未到账：诊断决策树 + 自助/工单分流
- [ ] 提现未到：诊断决策树 + 状态查询
- [ ] 2FA/登录异常：安全引导流 + 冻结账户入口
- [ ] KYC 问题：材料清单 + 申诉入口
- [ ] 反诈骗：渠道核验 + 高风险处理
- [ ] App/网页异常：技术排障 + 工单生成

### WP5：工单编排 + 追踪
- [ ] 工单类型识别
- [ ] 字段预填生成
- [ ] SLA 计算
- [ ] 进度查询接口
- [ ] 消息推送/提醒

### WP6：安全与权限
- [ ] AI 不执行高风险动作的硬拦截
- [ ] 操作审计日志
- [ ] 敏感操作二次确认
- [ ] 幻觉检测 / 置信度过滤

## 待确认事项（Decision Records）

### D1：技术栈
- 后端：FastAPI / LangChain / LlamaIndex？
- 前端：Next.js / 纯 API + 接入方自建？
- 向量数据库：Chroma（轻量）/ Elasticsearch（生产级）/ Pinecone（托管）？
- 知识库存储：飞书 Bitable + 原生文档 / 还是独立建设？

### D2：知识库数据源
- 现有帮助中心 CMS 是否有 API？
- 公告系统数据格式？
- 参数库是否有结构化数据源？

### D3：对接系统
- 工单系统用哪家？自建 / 现有系统 API？
- 用户账户状态接口权限范围？

### D4：部署形态
- SaaS 多租户 / 单租户私有化？
- 支持哪些交易所对接？（目前 spec 定位泛交易所，但 MVP 是否针对某一具体交易所？）

### D5：数据指标采集
- 上线第一天就要埋点的指标有哪些？
- 现有监控系统能否复用？

---

## Phase 1 验收标准

1. 用户提问"充值没到账" → 3轮内给出明确诊断 + 动作指引
2. 所有答复附带来源引用
3. 高风险动作（冻结/改密码）需用户明确确认
4. 工单预填字段完整率 ≥ 90%
5. RAG 检索 precision@3 ≥ 80%（人工抽检）
